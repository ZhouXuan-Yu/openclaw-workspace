# 多平台内容生成 Skill

## 触发条件
用户说"写一篇关于XXX的内容发到各平台"或"帮我写XXX的多平台文案"

## 工作流程

### 输入
- 主题/关键词
- 目标平台（可选，默认全部）
- 内容风格（可选，默认专业）

### 步骤

1. **分析主题**
   - 搜索相关信息（web_search）
   - 整理核心观点

2. **生成基础内容**
   - 写一篇 1500-2000 字的完整文章
   - 包含标题、正文、总结

3. **平台适配**
   根据各平台特点改写：

   | 平台 | 格式 | 字数 | 特点 |
   |------|------|------|------|
   | 小红书 | 图文笔记 | 300-800字 | emoji多、口语化、分段短、配图 |
   | 抖音 | 图文/视频脚本 | 200-500字 | 前3秒抓眼球、节奏快 |
   | 公众号 | 长文 | 1500-3000字 | 深度、排版、引导关注 |
   | 知乎 | 回答/文章 | 1000-2000字 | 专业、数据支撑、引用 |
   | CSDN | 技术博客 | 1500-3000字 | 代码块、技术深度 |
   | 掘金 | 技术文章 | 1000-2000字 | 实战、代码、流程图 |

4. **生成配图提示词**
   - 小红书封面图 prompt
   - 抖音封面图 prompt
   - 公众号头图 prompt

5. **输出**
   - 各平台内容写入 `memory/content/<日期>-<主题>/`
   - 每个平台一个文件：`<平台>.md`
   - 配图提示词：`image-prompts.md`
   - 汇总报告

### 输出格式
```
memory/content/2026-06-12-ai-agent/
├── xiaohongshu.md    # 小红书笔记
├── douyin.md         # 抖音图文
├── gongzhonghao.md   # 公众号文章
├── zhihu.md          # 知乎回答
├── csdn.md           # CSDN博客
├── juejin.md         # 掘金文章
├── image-prompts.md  # 配图提示词
└── summary.md        # 汇总（各平台标题+标签+摘要）
```

## 依赖
- web_search（搜索素材）
- image_generate（生成配图）
- write（写入文件）

## 质量标准
- 每个平台内容必须符合该平台风格
- 标题必须吸引眼球（非标题党）
- 包含相关标签/话题
- 小红书必须有 emoji
- 技术平台必须有代码示例（如适用）
