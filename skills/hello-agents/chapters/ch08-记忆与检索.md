# 第8章 记忆与检索

## Core Idea
借鉴人类记忆的分层结构(感觉→工作→长期),为 HelloAgents 增加记忆系统(Memory)和检索增强生成(RAG)。记忆解决对话遗忘问题，RAG 突破模型内置知识局限。

## Frameworks Introduced
- **工作记忆 / 情景记忆 / 语义记忆 / 感知记忆**: HelloAgents 的四层记忆类型
- **Qdrant 向量存储**: 高性能语义检索
- **Neo4j 图存储**: 知识图谱管理
- **SQLite 文档存储**: 结构化持久化
- **RAG 管道**: 文档处理器 + 嵌入层 + 向量检索

## Key Concepts
- **感觉/工作/长期记忆** — 人类记忆三层级(认知心理学)
- **无状态问题** — LLM 本身不记住对话历史
- **知识时效性** — LLM 训练数据有截止点
- **检索增强生成 (RAG)** — 在生成前检索外部知识作为上下文
- **Embedding** — 向量化表示用于语义检索
- **memory_tool** — HelloAgents 中记忆的统一 Tool 接口
- **rag_tool** — RAG 检索的统一 Tool 接口

## Mental Models
1. **人类记忆类比**: 工作记忆(短时) + 情景记忆(事件) + 语义记忆(知识)
2. **检索→生成 = 查资料→写报告**: 先搜后用，减少幻觉

## Anti-patterns
- 把所有历史塞进上下文导致上下文腐蚀
- 不分层存储(所有记忆混在一起)降低检索效率
- 忽略 TTL(生存时间)导致过期数据干扰当前决策

## Code Examples
```
from hello_agents import SimpleAgent, HelloAgentsLLM
from hello_agents.tools import MemoryTool
agent = SimpleAgent(name="学习助手", llm=HelloAgentsLLM())
agent.add_tool(MemoryTool())  # 添加记忆能力

# 第一次对话
response1 = agent.run("我叫张三，正在学Python")
# 第二次对话（需要记忆）
response2 = agent.run("你还记得我的名字吗？")  # "记得！你是张三"
```

## Worked Example
**解决无状态问题**: 
1. 第一次对话: agent.run("我叫张三") → 记忆存入工作记忆
2. 同一 session 的第二次对话: agent.run("你还记得我的学习进度吗？") → 从记忆系统中检索 → 回复"记得，你是张三，正在学 Python"
3. 使用 RAG tool 从外部文档库检索相关信息增强回答

## Key Takeaways
1. 智能体需要分层次记忆(工作/情景/语义)以应对不同场景
2. RAG 通过外部知识检索有效减少 LLM 幻觉
3. 记忆系统需要 TTL 管理和重要性排序
4. 向量检索 + 图检索互补(语义 vs 关系)
5. HelloAgents 将记忆和 RAG 抽象为 Tool，保持统一接口
