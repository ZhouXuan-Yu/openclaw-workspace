# 第6章 框架开发实践

## Core Idea
Agent 框架提供工程化规范，将通用组件抽象化。AutoGen/AutoScope/CAMEL/LangGraph 分别代表对话协作、工程平台、角色扮演、状态图四种路径。

## Frameworks Introduced
- **AutoGen (v0.7.4)**: 对话驱动多智能体协作，分层+异步架构
- **AgentScope**: 多智能体开发平台，强调易用性和工程化
- **CAMEL**: 角色扮演 (Role-Playing)，通过 Inception Prompting 自主对话
- **LangGraph**: Agent 执行流建模为状态图，天然支持循环

## Key Concepts
- **AssistantAgent** — AutoGen 的 LLM 驱动智能体
- **UserProxyAgent** — 代表用户+执行器，区分"思考"与"行动"
- **RoundRobinGroupChat** — 按序轮流发言的群聊协调机制
- **Team** — AutoGen 0.7.4 的灵活群聊概念
- **系统消息 (System Message)** — 智能体角色/职责/行为准则
- **异步优先 (async/await)** — 多智能体协作的并发处理
- **状态图 (State Graph)** — LangGraph 的节点=操作，边=跳转逻辑

## Mental Models
1. **"圆桌会议"**: 多智能体协作 = 专家组结构化讨论
2. **状态图即流程图**: 节点=操作，边=跳转逻辑，循环=自我反思

## Anti-patterns
- 系统消息太简单导致智能体行为失控
- 忽略异步性能，串行调用导致严重延迟
- 无终止条件，群聊可能无限对话

## Code Examples
```
# AutoGen 智能体定义
from autogen_agentchat.agents import AssistantAgent

def create_product_manager(model_client):
    return AssistantAgent(
        name="ProductManager",
        model_client=model_client,
        system_message="你是一位产品经理，请分析需求...完成后说'请工程师开始实现'"
    )
```

## Worked Example
**模拟软件开发团队**: ProductManager/Engineer/CodeReviewer/UserProxy 四个角色，RoundRobinGroupChat 编排，自动完成"开发比特币价格显示 Web 应用"。

## Key Takeaways
1. AutoGen 0.7.4 的分层+异步设计提升多智能体协作性能
2. 系统消息质量直接决定多智能体协作效果
3. LangGraph 状态图天然适合循环工作流
4. CAMEL 的 Role-Playing 降低流程设计复杂度
