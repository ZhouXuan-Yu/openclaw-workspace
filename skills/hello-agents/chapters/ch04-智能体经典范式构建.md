# 第4章 智能体经典范式构建

## Core Idea
三种经典范式从不同角度解决 Agent 的"思考-行动"问题：ReAct 边想边做，Plan-and-Solve 先谋后动，Reflection 通过自我批判修正结果。

## Frameworks Introduced
- **ReAct (Yao et al. 2022)**: Reasoning + Acting 结合，Thought-Action-Observation 循环
- **Plan-and-Solve (Lei Wang et al. 2023)**: 规划阶段 + 执行阶段解耦
- **Reflection**: 智能体通过自我批判和修正优化结果
- **HelloAgentsLLM**: 兼容 OpenAI 接口的通用 LLM 客户端
- **ToolExecutor**: 统一工具注册和执行管理器
- **SerpApi**: 结构化 Google 搜索结果 API

## Key Concepts
- **思考-行动-观察循环** — ReAct 核心交互模式
- **工具注册与描述** — 工具名+描述+执行逻辑，核心在描述质量
- **输出解析器** — 从原始文本提取 Thought 和 Action
- **最大步数限制** — 防止无限循环的安全阀
- **规划阶段/执行阶段** — Plan-and-Solve 的两阶段解耦
- **反思机制** — 自我批判+修正闭环

## Mental Models
1. **侦探 vs 建筑师**: ReAct 像侦探一步步推理，Plan-and-Solve 像建筑师先画蓝图
2. **"走一步看一步" vs "三思而后行"**
3. **自我批判循环**: 生成 → 批判 → 修正 → 再批判

## Anti-patterns
- 没有步数限制导致无限循环
- 工具描述模糊导致 LLM 无法正确选择
- 忽略原始输出：解析失败时务必打印原始文本
- temperature 过高导致输出不稳定，ReAct 需设为 0

## Code Examples
```
class HelloAgentsLLM:
    def __init__(self, model=None, apiKey=None, baseUrl=None, timeout=None):
        self.model = model or os.getenv("LLM_MODEL_ID")
        self.client = OpenAI(api_key=apiKey or os.getenv("LLM_API_KEY"),
                             base_url=baseUrl or os.getenv("LLM_BASE_URL"))
    def think(self, messages, temperature=0):
        response = self.client.chat.completions.create(
            model=self.model, messages=messages,
            temperature=temperature, stream=True)
        collected = [chunk.choices[0].delta.content or "" for chunk in response]
        return "".join(collected)
```

```
class ReActAgent:
    def run(self, question):
        for step in range(self.max_steps):
            prompt = REACT_PROMPT_TEMPLATE.format(tools=tools_desc, question=question, history="\n".join(self.history))
            response = self.llm_client.think([{"role":"user","content":prompt}])
            thought, action = self._parse_output(response)
            if action.startswith("Finish"):
                return re.match(r"Finish\[(.*)\]", action).group(1)
            tool_name, tool_input = self._parse_action(action)
            observation = self.tool_executor.getTool(tool_name)(tool_input)
            self.history.append(f"Action: {action}\nObservation: {observation}")
```

## Worked Example
**ReAct 回答"华为最新手机是什么"**: 第1步 Thought(需要搜索)→Action(Search[华为最新手机])→Observation(Pura 80 Pro+ / Mate 70 信息)→第2步 Thought(总结)→Action(Finish[...])。两步完成，展示动态规划能力。

## Key Takeaways
1. ReAct 高可解释性来自 Thought 链的透明展示
2. Plan-and-Solve 在强结构任务中保持更高目标一致性
3. Reflection 通过自我批判缓解幻觉，成本更高
4. 提示词脆弱性是所有范式共有痛点
5. 亲手实现范式是理解框架设计的必要条件
