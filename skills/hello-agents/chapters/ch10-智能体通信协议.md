# 第10章 智能体通信协议

## Core Idea
MCP 连接工具、A2A 连接智能体、ANP 构建网络。标准化接口替代手写专用适配器。

## Frameworks Introduced
- **MCP (Model Context Protocol, Anthropic)**: 智能体与工具标准化通信
- **A2A (Agent-to-Agent Protocol, Google)**: 智能体间点对点协作
- **ANP (Agent Network Protocol)**: 概念性框架，去中心化服务发现

## Key Concepts
- **上下文共享** — MCP 不只 RPC，还共享丰富上下文信息
- **对等通信** — A2A 中每个智能体既是服务者也是消费者
- **去中心化服务发现** — ANP 解决大规模网络中 Agent 互相发现

## Mental Models
1. **MCP = USB 协议**: 即插即用
2. **A2A = 团队沟通**: 专家间如何协作
3. **ANP = 互联网路由**: 大规模发现和连接

## Anti-patterns
- 为每个服务手写适配器
- 智能体间耦合过紧
- 无发现机制导致不可扩展

## Code Examples
```
# MCP 方式: 统一协议
from hello_agents.tools import MCPTool
github_mcp = MCPTool(server_command=["npx", "-y", "@modelcontextprotocol/server-github"])
agent.add_tool(github_mcp)  # 自动获得 GitHub 所有能力
```

## Worked Example
**集成多服务研究助手**: 通过 MCP 连接 GitHub、数据库、天气三个服务，统一通过 MCPTool 调用，不需要编写三个专用适配器。

## Key Takeaways
1. MCP 标准化工具通信，替代手写适配器
2. A2A 实现智能体间对等协作
3. ANP 解决大规模网络服务发现
4. 三种协议从工具→智能体→网络形成完整通信栈
