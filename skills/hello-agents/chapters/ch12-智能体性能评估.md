# 第12章 智能体性能评估

## Core Idea
智能体评估面临输出不确定性、标准多样、成本高昂三大挑战。系统化评估通过标准基准和量化指标衡量 Agent。

## Frameworks Introduced
- **BFCL**: Berkeley Function Calling Leaderboard，工具调用精度
- **GAIA**: Meta+HuggingFace，真实世界问题，三难度级别
- **ToolBench/API-Bank/AgentBench/WebArena**: 各有侧重的评估基准
- **ChatEval/SOTOPIA**: 多智能体协作评估

## Key Concepts
- **评估基准 (Benchmark)** — 统一数据集+指标+评分方法
- **AST 匹配** — BFCL 的工具调用精度评估
- **准精确匹配 (Quasi Exact Match)** — GAIA 的答案评估
- **LLM Judge** — LLM 作为评估器
- **Win Rate** — 两两对比评估
- **评估成本** — 全面评估可能数百元

## Mental Models
1. **评估 = 智能体体检**: 多维度量化指标
2. **基准 = 竞赛标准**: 统一标准下客观对比

## Anti-patterns
- 只用准确率(忽略效率/鲁棒性/协作)
- 数据污染
- 忽略评估成本

## Code Examples
```
from hello_agents.evaluation.benchmarks.bfcl import BFCLDataset, BFCLEvaluator
dataset = BFCLDataset()  # 1120+ 样本
evaluator = BFCLEvaluator(method="ast_match")
for sample in dataset:
    score = evaluator.evaluate(agent.run(sample.prompt), sample.ground_truth)
```

## Worked Example
**BFCL 工具调用评估**: 给 Agent 需调用多函数的任务，用 AST 匹配检查 Action 序列与标准调用一致性。覆盖 simple/multiple/parallel/irrelevance 四类别。

## Key Takeaways
1. 智能体评估比传统软件测试更复杂
2. BFCL 专注工具调用，GAIA 专注综合问题
3. 评估应覆盖准确率/效率/鲁棒性/协作
4. LLM Judge 是有效自动化评估方法
5. 系统化评估是 Agent 从实验到生产的必经之路
