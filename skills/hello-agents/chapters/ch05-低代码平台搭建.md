# 第5章 基于低代码平台的智能体搭建

## Core Idea
低代码平台将智能体构建从编码转向编排。Coze、Dify、n8n 分别代表零代码创作、开源平台、通用自动化三条路径。

## Frameworks Introduced
- **Coze (扣子)**: 字节跳动出品，零代码 Agent 构建，强生态集成
- **Dify**: 开源 LLM 应用开发平台，支持 RAG Pipeline、Agent 工作流
- **n8n**: 开源工作流自动化工具，数百个预置节点

## Key Concepts
- **工作流** — Coze 的"关卡通关路线图"
- **对话流** — Coze 的"NPC 对话通关"
- **插件** — Coze 的"角色技能卡"
- **知识库** — Coze 的"游戏百科全书"
- **发布渠道** — Coze 一键发布到微信/飞书/豆包
- **RAG Pipeline** — Dify 核心能力

## Mental Models
1. **"搭积木"式开发**: 可视化节点替代代码
2. **"角色+技能+知识"三角**: 角色设定+插件技能+知识库=智能体能力

## Anti-patterns
- 过度依赖平台锁定
- 忽略底层原理，遇到复杂场景无法调试

## Code Examples
```
# Coze 系统提示示例
你是一位资深且权威的科技媒体编辑。
## 工作流
1. 日报开头标注日期和署名
2. 每条新闻添加 Emoji
3. 所有内容必须与 AI/LLM/AIGC 相关
4. 每条目提供原始链接
```

## Worked Example
**用 Coze 构建"每日 AI 简报"**:
1. 添加 RSS/GitHub/arXiv 插件
2. 编写系统提示设定编辑角色
3. 编排工作流: 数据源→大模型处理→格式化输出
4. 测试并一键发布

## Key Takeaways
1. Coze 适合快速原型，插件生态是最大优势
2. Dify 适合企业级应用，开源可自建
3. n8n 适合 AI+业务系统集成
4. 低代码提供高层次抽象，不替代代码理解
