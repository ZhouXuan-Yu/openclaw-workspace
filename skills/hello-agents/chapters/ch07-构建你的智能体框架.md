# 第7章 构建你的智能体框架

## Core Idea
从零构建 HelloAgents 框架，核心理念"轻量教学 + 万物皆为工具"。所有高级概念(Memory/RAG/MCP/RL)统一抽象为 Tool，消除不必要抽象层。

## Frameworks Introduced
- **HelloAgents 框架**: Datawhale 社区自建教学用 Agent 框架
- **分层架构**: core/ → agents/ 层
- **万物皆为工具**: 除 Agent 外一切通过 Tool 实现

## Key Concepts
- **轻量级依赖** — 仅依赖 OpenAI SDK 和基础库
- **渐进式学习路径** — 分版本迭代，可 pip 安装历史版本
- **Tool 抽象** — 统一工具接口，所有扩展通过 Tool 实现
- **Agent 基类** — Agent 执行循环通用模板

## Mental Models
1. **"先体验，后实现"**: pip 安装体验→从零实现
2. **平替原则**: 用标准 API 而非自创抽象

## Anti-patterns
- 过度抽象 — 教学框架目标是透明
- 过早优化 — 代码可读性 > 性能微调
- 框架依赖绑架 — 商用框架 API 变更频繁

## Code Examples
```
from hello_agents import SimpleAgent, HelloAgentsLLM
agent = SimpleAgent(name="助手", llm=HelloAgentsLLM())
response = agent.run("你好")
```

```
hello-agents/
├── hello_agents/
│   ├── core/          # Agent基类, LLM, 消息, 配置
│   ├── agents/        # SimpleAgent, ReActAgent, ReflectionAgent
│   └── tools/         # 内置工具
├── tests/
└── pyproject.toml
```

## Worked Example
**实现 SimpleAgent**: 定义 HelloAgentsLLM → Agent 基类(run 入口) → SimpleAgent 继承基类 + system_prompt → 用户输入后构造 messages → LLM 生成响应返回。

## Key Takeaways
1. 自建框架是从"使用者"到"构建者"的关键转变
2. "万物皆为工具"大幅降低认知负载
3. 框架应在功能完整性和学习友好性间平衡
4. 掌握设计原理后可在任何框架上深度定制
