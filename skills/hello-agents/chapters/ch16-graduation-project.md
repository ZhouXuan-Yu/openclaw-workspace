# 第16章 毕业设计

## Core Idea
从学习者到构建者——通过完整毕业设计项目整合全书知识，通过开源协作提交到 Hello-Agents 共创仓库。

## Frameworks Introduced
- **Git/GitHub**: 版本控制和开源协作
- **Jupyter Notebook**: 开发环境
- **Co-creation-projects 仓库**: Datawhale 共创仓库

## Key Concepts
- **项目选题** — 生产力/学习/创意/数据/生活五类
- **PR 提交** — GitHub Pull Request 流程
- **评审流程** — 社区 Review 并合并
- **项目结构** — README+requirements.txt+main.ipynb+src/
- **文档完整性** — 问题分析/功能/技术栈/示例

## Mental Models
1. **"开源即面试"**: PR 是你的技术作品集
2. **"学以致用"**: 全书知识选择性整合

## Anti-patterns
- 选题过大
- 忽略 README
- 不遵循社区规范
- 只想不写

## Code Examples

jjyaoao-CodeReviewAgent/

## Worked Example
**选题: CodeReviewAgent**: 问题分析→核心功能(代码质量/漏洞/性能/安全)→技术栈(HelloAgents+ReAct+Search)→交付 Notebook→GitHub PR 提交。

## Key Takeaways
1. 毕业设计是全书知识综合运用
2. 选题应实用且可完成
3. 开源协作是技术成长最佳路径
4. README 质量=项目质量
5. 完成比完美更重要
