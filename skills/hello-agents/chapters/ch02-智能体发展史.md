# 第2章 智能体发展史

## Core Idea
智能体从符号主义的逻辑推理出发，经历专家系统、分布式智能、学习范式三个重大转折，最终汇聚到 LLM 驱动的新时代。每一代范式都为了解决上一代的根本局限。

## Frameworks Introduced
- **物理符号系统假说 (PSSH)**: Newell & Simon (1976) — 智能的本质是符号的计算与处理
- **专家系统**: MYCIN、DENDRAL — 知识库+推理机的架构
- **产生式规则**: IF-THEN 规则 + 正向链/反向链推理
- **ELIZA**: Weizenbaum (1966) — 模式匹配+文本替换的聊天机器人
- **SHRDLU**: Winograd (1968-1970) — "积木世界"中的综合智能体

## Key Concepts
- **符号主义 (Symbolicism)** — 智能 = 符号操作 + 逻辑规则
- **物理符号系统假说 (PSSH)** — 通用智能行为的充分必要条件
- **知识库 / 推理机** — 专家系统的两大核心组件
- **正向链 / 反向链** — 数据驱动 vs 目标驱动的推理方式
- **置信因子 (Certainty Factor)** — MYCIN 引入的不确定性处理方法
- **知识获取瓶颈** — 手工编码知识的不可规模化困境
- **框架问题 (Frame Problem)** — 动态世界中高效判断不变量的逻辑难题
- **常识问题 (Common-sense Problem)** — Cyc 项目数十年的未竟挑战
- **模式匹配 / 文本替换** — ELIZA 的核心算法机制
- **亚符号主义 (连接主义)** — 神经网络的分布式知识表示
- **强化学习 (RL)** — 通过试错最大化长期收益的学习范式

## Mental Models
1. **问题驱动迭代**: 每一代范式解决上一代的痛点，同时引入新局限
2. **知识获取瓶颈**: 手工知识工程的边际成本递增，最终无法扩展
3. **微观世界方法**: SHRDLU 证明在简化环境中验证复杂 AI 原理的有效性

## Anti-patterns
- **过度依赖规则完备性**: 为现实世界编写完备规则几乎不可能
- **忽视框架问题**: 认为只需说明"什么变了"，不考虑"什么没变"的计算成本
- **把简单模式匹配当作真正理解**: ELIZA 效应

## Code Examples
```
import re, random
rules = {
    r'I need (.*)': ["Why do you need {0}?", "Would it really help you to get {0}?"],
}
def generate_response(user_input):
    for pattern, responses in rules.items():
        match = re.search(pattern, user_input, re.IGNORECASE)
        if match:
            return random.choice(responses).format(match.group(1))
    return "Please go on."
```

## Worked Example
**构建 mini-ELIZA**: 定义关键词规则 → 对输入匹配优先级最高的规则 → 捕获捕获内容 → 随机选择重组规则 → 代词转换。用户说 "I need a friend" → 返回 "Why do you need a friend?"

## Key Takeaways
1. 符号主义证明了"知识+推理"可以产生专业级智能(MYCIN 超越非专业医生)
2. 手工知识工程的不可扩展性最终成为瓶颈
3. 亚符号主义通过数据驱动学习突破了知识获取瓶颈
4. ELIZA 揭示: 简单的模式匹配足以制造"智能"假象
5. SHRDLU 的"感知-思考-行动"闭环设计至今仍是智能体基础
