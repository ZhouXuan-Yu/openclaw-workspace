# 第1章 初识智能体

## Core Idea
智能体是通过传感器感知环境、通过执行器采取行动以达成目标的自主实体。LLM 智能体将推理与工具调用结合，通过 Thought-Action-Observation 循环与外部世界交互。

## Frameworks Introduced
- **PEAS 模型**: Performance/Environment/Actuators/Sensors — 精确描述任务环境的四要素
- **ReAct 范式**: Yao et al. 2022 — 将 Reasoning 与 Acting 结合的经典范式
- **Thought-Action-Observation 循环**: 智能体核心交互协议

## Key Concepts
- **智能体 (Agent)** — 感知→决策→行动的自主实体
- **传感器 (Sensors) / 执行器 (Actuators)** — 感知能力与行动能力的物理/数字接口
- **自主性 (Autonomy)** — 基于感知和内部状态独立决策的能力
- **世界模型 (World Model)** — 智能体内部对环境的表征
- **反射智能体 / 基于模型的智能体 / 基于目标的智能体 / 基于效用的智能体** — 传统智能体类型阶梯
- **学习型智能体** — 通过 RL 从经验中自我改进的智能体
- **符号主义 / 亚符号主义 / 神经符号主义** — 三种知识表示范式
- **反应式 / 规划式 / 混合式** — 基于时间维度的分类
- **Workflow vs Agent** — 预设指令 vs 自主决策的关键差异

## Mental Models
1. **PEAS 模型**: 设计任何智能体前，先明确 Performance, Environment, Actuators, Sensors
2. **System 1 / System 2 (卡尼曼)**: 快速直觉 vs 慢速推理 — 神经符号主义的认知类比
3. **Agent Loop**: 感知→思考(规划+工具选择)→行动→观察 的闭环

## Anti-patterns
- **将 Agent 当作 Workflow 使用**: 给 Agent 写死 if-then 规则，扼杀了其自主推理能力
- **忽略环境不确定性**: 假设工具调用总是返回确定结果，不处理重试/错误
- **一次性给出过多指令**: 期望 Agent 一步完成所有任务，而不使用迭代循环

## Code Examples

```
# 指令模板定义
AGENT_SYSTEM_PROMPT = """
你是一个智能旅行助手。你的任务是分析用户的请求，并使用可用工具一步步地解决问题。
# 可用工具:
- `get_weather(city: str)`: 查询指定城市的实时天气。
- `get_attraction(city: str, weather: str)`: 根据城市和天气搜索推荐的旅游景点。
# 输出格式要求:
你的每次回复必须严格遵循以下格式，包含一对Thought和Action：
Thought: [你的思考过程和下一步计划]
Action: [你要执行的具体行动]
Action的格式必须是以下之一：
1. 调用工具：function_name(arg_name="arg_value")
2. 结束任务：Finish[最终答案]
"""
```

```
# 主循环核心
for i in range(5):
    llm_output = llm.generate(full_prompt, system_prompt=AGENT_SYSTEM_PROMPT)
    prompt_history.append(llm_output)
    action_match = re.search(r"Action: (.*)", llm_output, re.DOTALL)
    action_str = action_match.group(1).strip()
    if action_str.startswith("Finish"):
        final_answer = re.match(r"Finish\[(.*)\]", action_str).group(1)
        print(f"任务完成: {final_answer}")
        break
    # 解析工具名和参数
    tool_name = re.search(r"(\w+)\(", action_str).group(1)
    kwargs = dict(re.findall(r'(\w+)="([^"]*)"', args_str))
    observation = available_tools[tool_name](**kwargs)
    prompt_history.append(f"Observation: {observation}")
```

## Worked Example
**问题**: 构建一个能回答"今天北京的天气并推荐景点"的智能体。
1. 定义 get_weather(city) 和 get_attraction(city, weather) 两个工具
2. 编写 system prompt 定义角色和格式
3. 运行循环: LLM 输出 Thought → Action(get_weather) → Observation → Thought → Action(get_attraction) → Observation → Thought → Finish
4. 输出: "今天北京天气晴朗，适合游览颐和园"

## Key Takeaways
1. Agent 的核心是自主决策循环，不是预设规则
2. LLM 驱动使 Agent 能处理模糊、高层级的自然语言指令
3. 环境定义(PEAS)决定了智能体的设计边界
4. Workflow 适合确定路径，Agent 适合不确定路径
5. 神经符号主义融合了直觉(亚符号)与理性(符号)两种范式
