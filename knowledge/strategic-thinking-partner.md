# 战略思维伙伴 (Strategic Thinking Partner) 框架集

> 来源: McKinsey / BCG / Bain / 经典战略理论
> 整理日期: 2026-03-23

---

## 一、什么是好的战略思维伙伴

好的战略思维伙伴不是给答案的人，而是**帮你把问题想清楚的人**。核心能力：

1. **定义正确的问题** — 80% 的战略失败源于解错了题
2. **结构化拆解** — 把模糊的"怎么办"变成可分析的子问题
3. **挑战假设** — 指出你思维中未经检验的前提
4. **连接信息** — 在碎片中看到模式
5. **推动行动** — 思考的终点是决策，不是分析

## 二、McKinsey 框架体系

### 2.1 问题解决七步法 (McKinsey 7-Step Problem Solving)

| 步骤 | 核心动作 | 关键产出 |
|------|----------|----------|
| 1. 定义问题 | 用 MECE 原则界定问题边界 | 问题陈述书 |
| 2. 拆解问题 | 逻辑树 / Issue Tree | 可独立验证的子问题 |
| 3. 优先排序 | 80/20，砍掉不重要的分支 | 优先级排序 |
| 4. 制定计划 | 工作流 + 分析计划 | 工作计划表 |
| 5. 执行分析 | 数据收集 + 假设验证 | 分析结果 |
| 6. 综合发现 | 合成 (Synthesis) | So-What |
| 7. 沟通传达 | 金字塔原理 | 一页纸 / 幻灯片 |

### 2.2 MECE 原则 (Mutually Exclusive, Collectively Exhaustive)

- **互斥**: 分类之间无重叠
- **穷尽**: 合起来覆盖全貌
- 实操: 每次拆解问自己"有没有遗漏？有没有重叠？"

### 2.3 假设驱动 (Hypothesis-Driven Approach)

```
不要: 搜集所有数据 → 分析 → 得出结论
要:   提出假设 → 设计验证 → 数据确认/推翻 → 修正假设
```

核心价值: 避免"分析瘫痪"，聚焦于能改变决策的信息。

### 2.4 金字塔原理 (Pyramid Principle, Barbara Minto)

- **结论先行**: 先说答案，再展开论据
- **纵向对应**: 每层论据支撑上层论点
- **横向逻辑**: 同层论据之间 MECE
- 三种结构: 归纳 / 演绎 / 演绎+归纳混合

### 2.5 SCQA 沟通框架

| 要素 | 含义 | 示例 |
|------|------|------|
| **S**ituation | 背景共识 | "我们的市场份额连续3季度下滑" |
| **C**omplication | 冲突/矛盾 | "但行业整体增长了15%" |
| **Q**uestion | 自然引出的问题 | "增长去哪了？我们为什么掉队？" |
| **A**nswer | 你的建议 | "核心问题是渠道下沉不足" |

### 2.6 三层面增长理论 (Three Horizons of Growth)

| 层面 | 含义 | 时间跨度 | 管理重点 |
|------|------|----------|----------|
| H1 | 核心业务 | 0-1年 | 利润最大化、效率 |
| H2 | 新兴业务 | 1-3年 | 规模化、市场验证 |
| H3 | 未来种子 | 3-10年 | 探索、实验、期权 |

**关键**: 三个层面必须同时管理，不能只顾 H1。

## 三、BCG 框架体系

### 3.1 BCG 矩阵 (Growth-Share Matrix)

```
          高市场份额          低市场份额
高增长  | ★ 明星 (Star)     | ? 问题 (Question Mark) |
低增长  | 💰 现金牛 (Cash Cow) | 🐕 瘦狗 (Dog)        |
```

策略:
- 明星: 持续投资
- 现金牛: 收割利润，投资其他
- 问题: 要么大投入做成明星，要么退出
- 瘦狗: 退出/收割

### 3.2 BCG 经验曲线 (Experience Curve)

- 累计产量每翻一倍，单位成本下降 20-30%
- 战略含义: 先行者可以通过规模获得成本优势
- 适用于: 标准化产品、规模效应明显的行业

### 3.3 BCG 时间竞争 (Time-Based Competition)

- 缩短从概念到现金的周期
- 不是"更快地做同样的事"，而是"做不同的事"
- 关键指标: 时间-价值比率

### 3.4 BCG 战略调色板 (Strategy Palette, Martin Reeves)

5 种战略风格，根据环境可预测性/可塑性选择:

| 环境 | 可预测 | 不可预测 |
|------|--------|----------|
| **可塑造** | 适应型 (Adaptive) | 塑造型 (Shaping) |
| **不可塑造** | 经典型 (Classical) | 重塑型 (Renewal) |
| **两者之间** | — | 愿景型 (Visionary) |

**核心洞察**: 没有万能战略，战略风格必须匹配环境特征。

## 四、Bain 框架体系

### 4.1 客户结果导向 (Results Delivery)

Bain 的核心理念: 咨询的价值 = 实际结果，不是报告。

- **Results Delivery**: 不是"交付建议"，是"交付结果"
- 实操: 每个项目定义可衡量的结果指标，全程追踪

### 4.2 忠诚度经济学 (Loyalty Economics)

- 核心发现: 客户留存率提高 5%，利润增长 25-95%
- **净推荐值 (NPS)**: 衡量客户忠诚度的关键指标
- 战略含义: 以客户忠诚度为核心驱动增长

### 4.3 优势矩阵 (Advantage Matrix)

```
        少数优势          多数优势
大体量 | 专业化 (Specialization) | 规模化 (Scale) |
小体量 | 细分 (Segmentation)    | 碎片化 (Fragmented) |
```

决定竞争策略的关键: 优势的数量和体量的大小。

### 4.4 决策有效性 (Decision Effectiveness)

Bain 研究发现，决策质量 = 速度 × 准确性 × 杀伤力(执行力度)

高绩效组织的决策特征:
- 决策少而精（不试图做所有决定）
- 明确谁决策、谁输入、谁执行
- 快速决策 + 快速修正

## 五、经典战略分析框架

### 5.1 Porter 五力模型 (Porter's Five Forces)

```
                    新进入者威胁
                         ↓
  供应商议价力 ← 行业竞争强度 → 买方议价力
                         ↑
                    替代品威胁
```

- 分析维度: 每个力的强度和趋势
- 战略含义: 在某个力薄弱处建立位置

### 5.2 Porter 三大通用战略

| 战略 | 核心 | 风险 |
|------|------|------|
| 成本领先 | 最低成本生产 | 被技术颠覆 |
| 差异化 | 独特价值 | 溢价无法维持 |
| 聚焦 | 细分市场专精 | 细分被侵蚀 |

**关键洞察**: 必须选一个。"夹在中间"是最差位置。

### 5.3 SWOT + TOWS 矩阵

SWOT 只是起点，TOWS 才有战略含义:

|  | 机会 (O) | 威胁 (T) |
|--|----------|----------|
| **优势 (S)** | SO: 进攻策略 | ST: 防御策略 |
| **劣势 (W)** | WO: 改进策略 | WT: 生存策略 |

### 5.4 价值链分析 (Value Chain)

**主要活动**: 进料物流 → 生产 → 出料物流 → 营销 → 服务
**支持活动**: 企业基础设施 / 人力资源 / 技术开发 / 采购

战略含义: 在某个环节建立不可替代的优势。

### 5.5 PESTEL 分析

| 因素 | 关注点 |
|------|--------|
| **P**olitical | 政策、法规、政治稳定性 |
| **E**conomic | 增长率、利率、汇率、通胀 |
| **S**ocial | 人口、文化、消费行为 |
| **T**echnological | 技术变革、创新速度 |
| **E**nvironmental | 环保、可持续性 |
| **L**egal | 法律框架、合规要求 |

## 六、思维模型 (Mental Models for Strategic Thinking)

### 6.1 第一性原理 (First Principles Thinking)

- 回到最基本的事实和物理约束
- 从那里重新推导，而不是类比
- 典型应用: Elon Musk 的 SpaceX 成本分析

### 6.2 二阶思维 (Second-Order Thinking)

- 不只看直接结果，要看连锁反应
- 问: "然后呢？再然后呢？"
- 避免: 只优化一阶效果，忽略二阶副作用

### 6.3 反演法 (Inversion)

- 不问"怎样成功"，问"怎样一定失败"
- 然后避免那些失败因素
- Charlie Munger: "反过来想，总是反过来想"

### 6.4 能力圈 (Circle of Competence)

- 知道自己知道什么，更重要的是知道自己不知道什么
- 战略含义: 在能力圈内做决策，持续扩展边界

### 6.5 安全边际 (Margin of Safety)

- 留出缓冲空间应对不确定性
- 战略含义: 不要all-in在最乐观的场景上

## 七、战略咨询师的核心工作模式

### 7.1 好的战略思维伙伴会做什么

1. **问对问题** — 不急于给答案，先确保问题定义正确
2. **挑战共识** — "大家都这么认为"恰恰需要质疑
3. **区分信号与噪音** — 哪些数据真正改变决策？
4. **强制排序** — 不能什么都重要，必须有优先级
5. **连接 dots** — 在看似无关的信息中找到关联
6. **制造不舒服** — 好的战略讨论应该让人不适（因为要面对残酷现实）
7. **推动行动** — 每个分析都必须指向"所以呢？我们该怎么做？"

### 7.2 常见的战略思维陷阱

| 陷阱 | 表现 | 对策 |
|------|------|------|
| 确认偏误 | 只找支持自己的数据 | 主动寻找反面证据 |
| 锚定效应 | 被第一个数字锁住 | 多角度估算 |
| 沉没成本 | 因为已经投入所以继续 | 只看未来增量 |
| 分析瘫痪 | 永远在分析不决策 | 设定决策截止日 |
| 群体思维 | 团队不敢反对 | 指定"魔鬼代言人" |
| 框架依赖 | 只会用已知工具 | 先思考再选框架 |

### 7.3 "So What" 测试

每个分析完成后问三个问题:
1. **So what?** — 这意味着什么？
2. **What if?** — 如果不这样呢？
3. **What now?** — 下一步做什么？

## 八、战略思维伙伴的对话模式

### 8.1 结构化对话五步

```
[1] 澄清 — "你说的X具体指什么？"
[2] 拆解 — "这个问题可以分成哪几个子问题？"
[3] 质疑 — "这个假设的依据是什么？"
[4] 综合 — "所以这意味着什么？"
[5] 推进 — "基于此，下一步是什么？"
```

### 8.2 战略对话的提问清单

- 你真正想要解决的问题是什么？（不是表面问题）
- 如果只能改变一件事，你会改什么？
- 你的竞争对手会怎么反应？
- 什么情况下你的判断会是错的？
- 你有什么证据能推翻自己的结论？
- 10年后回看，这个决策会怎样？
- 最大的风险是什么？你能承受吗？
