# AI Agent + 教培项目 · 决策记录
> 2026-06-16 第一次讨论

---

## 一、项目定位

**一句话**：让每个教培老师拥有一个"越用越懂学生"的 AI 教学搭档。

**核心理念**：AI 不替代老师，而是让一个老师能同时关注到每个学生。

**不是**：ChatGPT 套壳、学生直接面对的聊天机器人、一次性工具
**是**：教师的倍增器、越用越有价值的飞轮、长期产品

---

## 二、目标用户

### 第一阶段：个人教师（MVP）
- 独立教培老师
- 目标：跑通核心链路，验证需求
- 模式：免费/低价内测

### 第二阶段：中小教培机构
- 50-500 学生规模
- 目标：定制化服务，积累行业知识
- 模式：定制费 + 月费

### 用户画像
- 决策人：老师自己 / 机构老板
- 付费意愿：机构 > 个人
- 核心痛点：课后答疑耗时、学情靠感觉、教案重复劳动
- 触达方式：微信生态（小程序 + 公众号 + 企微）

---

## 三、学科切入点

**数学**，原因：
- 答案固定，客观题可自动批改
- 知识点依赖关系清晰（一元二次方程 → 二次函数 → 不等式）
- 难度分级明确，可出 5 级梯度
- 效果可量化（成绩提升是最硬指标）

---

## 四、产品架构

### 双端架构

```
教师端 → Web 网站（浏览器访问）
├── 学情看板
├── 教案助手
├── 练习生成
├── 学生档案管理
└── 消息中心

学生/家长端 → 微信小程序
├── 做题
├── 错题本
├── AI 答疑对话
├── 学情报告查看
└── 家长：成长趋势
```

### 为什么不全做小程序
- 教师端功能复杂，小程序屏幕小、体验差
- 学情看板需要大屏展示
- 教师日均使用 2-3 小时，小程序不适合长时间操作

### 为什么不全做独立 App
- 学生获客成本极高（要下载）
- 家长不愿给孩子装更多 App
- 双端开发成本 ×2
- 迭代要走应用商店审核

### 技术栈

| 层 | 选型 |
|---|------|
| AI 引擎 | DeepSeek V3 / 通义千问（国产模型，便宜） |
| Agent 框架 | LangChain / LangGraph |
| 后端 | Python FastAPI |
| 教师端 | Next.js（Web） |
| 学生端 | 微信小程序（Taro / uni-app） |
| 数据库 | PostgreSQL（装服务器上） |
| 向量库 | Qdrant（知识检索） |
| 缓存 | Redis |
| 存储 | 阿里云 OSS（教案/课件文件） |
| 部署 | 阿里云轻量服务器 |

## 五、六大 Agent

| Agent | 功能 | 自进化点 |
|-------|------|---------|
| 学情分析 | 成绩/错题/行为自动采集，生成报告 | 追踪预警 → 干预 → 效果闭环 |
| 教案生成 | 输入知识点 → 输出教案 + PPT 大纲 | 学习教师修改模式，减少修改量 |
| 练习生成 | 按知识点+难度出题，错题变式 | 积累正答率模型，优化难度预测 |
| 答疑辅导 | 学生微信端提问，苏格拉底式引导 | 追踪哪种引导方式对哪类学生有效 |
| 跟进提醒 | 作业/成绩/续费自动提醒 | 优化提醒频率和话术 |
| 个性化推荐 | 学习路径 + 资源推荐 | 追踪推荐效果，自动择优 |

## 六、学生档案系统

### 结构
```
基础信息 → 年级/学校/联系方式
学业画像 → 知识点掌握度/成绩趋势/错题本（自动采集）
行为画像 → 拖延型/粗心型/畏难型/冲刺型（Agent 自动打标）
个性化策略 → AI 推荐 + 教师确认
沟通记录 → 家长沟通摘要/教师备注
长期追踪 → 阶段对比/成长报告/效果归因
```

### 关键设计
- 自动采集为主，教师手动为辅
- 档案"活"机制：每次互动自动更新
- 家长每月收到成长报告（自动从档案生成）
- 隐私红线：数据不外传、不训练外部模型

---

## 七、自进化引擎

### 三层进化
```
单教师进化 → 学会张老师的教案风格，修改量 60%→5%
跨教师进化 → 王老师的类比法有效 → 推荐给李老师
跨机构进化 → 5000 学生数据 → 行业级教学洞察
```

### 落地节奏（务实版）
```
第 1 个月：纯记录
  → 记录教师修改、学生成绩，不做优化

第 2 个月：简单规则
  → 教师总把难度改成"简单"→ 系统记住默认用"简单"

第 3 个月：模式识别
  → "计算类错误"做 3 道变式题后正确率提升 40%→ 提取为策略

第 6 个月：策略沉淀
  → 机构积累 50+ 有效教学策略，新老师直接获得
```

### 安全边界
- 可以进化：教案风格、练习难度、答疑话术
- 不能进化：不改大纲、不绕过教师审核、不外传数据

---

## 八、护城河

```
第 1 层：教师迁移成本（教了 AI 3 个月，换平台要重教）
第 2 层：机构知识沉淀（教学策略库在系统里，换系统归零）
第 3 层：数据飞轮（更多使用 → 更好 AI → 更多使用）
第 4 层：行业知识图谱（跨机构教学效果数据）
```

**核心逻辑**：用得越久越离不开，不是工具，是会成长的教学搭档。

## 九、商业模式

### 定价
| 版本 | 月费 | 包含 |
|------|------|------|
| 体验版 | ¥0 | 教案 20 次/月 + 基础学情 |
| 专业版 | ¥299/月 | 全部功能 + 50 学生 + 自进化 |
| 机构版 | ¥899/月 | 多教师 + 无限学生 + 策略共享 |

### 战略路径
```
阶段一（1-2 月）：个人教师 MVP，跑通链路
阶段二（3-6 月）：2-3 家机构定制化服务
阶段三（6-12 月）：产品化，标准 SaaS + 可选定制
```

### 为什么走定制化
- 大厂做不了定制（要规模化）
- 每个机构都是"案例"和"数据源"
- 定制费覆盖成本，不靠融资
- 定制经验抽象后就是产品

## 十、成本控制

### AI 模型选择
- **不用 OpenAI/Claude**（贵、需海外支付）
- **用 DeepSeek V3 / 通义千问**（便宜 50-100 倍、人民币结算、国内直连）
- **80% 场景用小模型或规则引擎**，只有复杂任务调大模型

### 模型分层调用
| 场景 | 模型 | 成本 |
|------|------|------|
| 客观题批改 | 代码直接判 | ¥0 |
| 教案生成 | DeepSeek/Qwen | 极低 |
| 学生答疑 | 中等模型 + 知识库检索 | 低 |
| 学情分析 | 规则 + 统计 | ¥0 |

### 成本估算
| 阶段 | 月成本 |
|------|--------|
| 开发测试 | ¥0-30（免费额度） |
| 种子期（1 机构 50 学生） | ¥150-300 |
| 成长期（10 机构 500 学生） | ¥800-1500 |
| 规模期（100 机构） | ¥5000-10000 |

### 省钱策略
- 服务器：阿里云轻量服务器 ¥60/月
- 数据库：直接装服务器上，不买云数据库
- 存储：OSS 按量付费
- 新用户优惠：阿里云/腾讯云首年优惠

## 十二、MVP 路线图

### Phase 1（第 1-3 周）：核心链路
```
教师 Web 端：教案生成 + 练习生成
学生小程序：做题 + 查看解析
学生档案 V1：成绩趋势 + 错题本
教师行为日志采集（为自进化储备数据）
```

### Phase 2（第 4-6 周）：智能答疑
```
微信小程序接入答疑 Agent
引导式对话（苏格拉底式）
学生档案 V2：行为标签 + 对话记录
答疑效果追踪
```

### Phase 3（第 7-9 周）：学情闭环 + 家长触达
```
学情报告自动生成
家长端成长报告
跟进提醒 Agent
简单自进化（教师偏好学习）
```

### Phase 4（第 10-14 周）：多机构 + 策略沉淀
```
机构级教学策略库
多机构数据隔离
个性化推荐 Agent
效果追踪闭环
```

### 一周内能跑通的最小 Demo
```
Day 1-2：教师端 Web + DeepSeek API → 输入知识点 → 生成教案+练习
Day 3-4：学生 H5 做题页面 → 扫码做题 → 自动批改 → 解析
Day 5-6：学生档案 V1 → 成绩入库 → 错题本 → 趋势图
Day 7：部署到阿里云 → 找 1 个老师内测
总投入：¥60 服务器 + ¥0 API
```

## 十三、风险与对策

| 风险 | 对策 |
|------|------|
| AI 内容质量不稳 | 教师审核不可跳过 |
| 教师不愿教 AI（冷启动） | 开箱即用模板，降低门槛 |
| 学生过度依赖 AI | 引导式设计 + 每日次数限制 |
| 大厂入场 | 靠数据飞轮和迁移成本建壁垒 |
| API 成本上涨 | 模型切换能力 + 数据积累后微调 |
| 数据安全 | 加密存储 + 定期审计 + 合规先行 |
