# ADR-0001: 记忆4层架构

**状态**: 已接受
**日期**: 2026-06-11
**决策者**: ZhouXuan + GGOB

## 背景

OpenClaw 需要一个持久化记忆系统来跨会话保持上下文。最初只有 MEMORY.md 一个文件，随着使用增长，需要分层管理。

## 决策

采用4层记忆架构：

| 层 | 内容 | 加载策略 |
|----|------|---------|
| L1 索引 | MEMORY.md (<200行) | 每次必读 |
| L2 主题 | memory/topics/ | L1命中按需读 |
| L3 日志 | memory/daily/ | 读今日（不自动加载昨日） |
| L4 会话 | sessions/*.jsonl | 最后手段 |

## 理由

- **L1 保持小**: MEMORY.md <200行，确保每次会话都能完整加载
- **L2 按需**: 主题文件只在相关时加载，避免信息过载
- **L3 只读今日**: 每日日志只加载当天，历史按需检索
- **L4 最后手段**: 会话日志是最详细的，但也是最占资源的

## 后果

- 写入规则：明确指令→daily⭐+MEMORY.md+topic；纠正→首次daily第2次topic
- 单次不提权，纠正权重×5，低密度丢弃
- 需要维护 _graph.json 做主题关联图
