## 一、核心定位

**一句话**：让每个教培老师拥有一个"越用越懂学生"的 AI 教学搭档。

**关键差异**：
- 不是 ChatGPT 套壳，是一个**会学习的系统**
- 不是学生直接面对的机器人，是**教师的倍增器**
- 不是一次性工具，是**越用越有价值的飞轮**

## 三、六大 Agent 详解

### Agent 1：学情分析 Agent

**输入**：作业/测试/课堂互动/AI 对话记录
**输出**：个人学情报告 + 班级分析 + 预警

```
功能清单：
├── 自动记录每次练习的正确率、用时、错题类型
├── 生成知识图谱掌握度（哪些知识点薄弱）
├── 异常检测：连续下降 / 偏离均值 / 拖延模式
├── 周报/月报自动生成（可发给家长）
├── 班级横向对比（匿名化处理）
└── 学习风格识别（视觉型/听觉型/动手型）
```

**自进化点**：
- 追踪"AI 预警 → 教师干预 → 学生变化"的完整闭环
- 积累哪些预警模式真正有效，淘汰误报率高的模式

### Agent 2：教案生成 Agent

**输入**：学科/年级/教学目标/课时数 + 学生档案数据
**输出**：个性化教案 + PPT 大纲 + 课堂活动

```
功能清单：
├── 根据教学大纲自动生成教案框架
├── 结合班级学情数据调整难度和重点
├── 生成课堂互动环节（提问/讨论/小测）
├── 支持教师自定义模板和风格
├── 教案版本管理 + 复用
└── 引用教学策略库中的有效方法
```

**自进化点**：
- 教师每次修改 AI 生成的教案，系统记录修改模式
- 追踪教案使用后的学生表现，建立"教案特征 → 教学效果"映射
- 积累后：AI 生成的教案越来越接近教师个人风格

### Agent 3：练习生成 Agent

**输入**：知识点/难度/题型/数量 + 学生档案
**输出**：个性化练习 + 答案解析

```
功能清单：
├── 按知识点 + 难度梯度生成练习
├── 基于学生错题本自动生成针对性练习
├── 支持多种题型（选择/填空/解答/作文）
├── 自动批改（客观题出分，主观题评分建议）
├── 错题归档 + 变式题推荐
└── 基于历史数据预测"这道题学生大概率错在哪"
```

**自进化点**：
- 追踪每道题的正答率、常见错误选项
- 积累"某知识点 + 某难度"的真实正答率模型
- 题目质量评估：低区分度的题自动降权

### Agent 4：答疑辅导 Agent（学生侧）

**触发**：学生在微信端提问
**行为**：引导式答疑，个性化深度适配

```
对话策略（基于学生档案自动选择）：
├── 畏难型 → 多鼓励，拆更细，降低每步难度
├── 粗心型 → 不直接说答案，引导自己检查
├── 冲刺型 → 直接给思路，不啰嗦
├── 基础薄弱型 → 回退到前置知识点
└── 教师可自定义策略
```

**自进化点**：
- 追踪每次答疑后学生是否真正掌握（后续同类题正答率）
- 积累"哪种引导方式对哪类学生最有效"
- 对话质量评估：学生主动追问 vs 沉默离开

### Agent 5：跟进提醒 Agent

**触发**：定时任务 / 事件驱动
**行为**：自动跟进 + 精准提醒

```
触发场景：
├── 学生 3 天未完成作业 → 微信提醒（话术根据学生档案个性化）
├── 连续 2 次测试下降 → 通知教师 + 建议干预方案
├── 课程到期前 7 天 → 续费提醒（附学生成长数据）
├── 上课前 1 小时 → 课程提醒
├── 学生提问后 24h 未回复 → 通知教师
└── 检测到学生情绪低落 → 提醒教师关注
```

**自进化点**：
- 追踪提醒后的行为变化（提醒了是否真的去做了）
- 优化提醒频率和话术（太多会烦，太少没用）

### Agent 6：个性化推荐 Agent（新增）

**输入**：学生档案 + 学习历史 + 当前状态
**输出**：个性化学习路径 + 资源推荐

```
功能清单：
├── 基于知识图谱推荐下一步学习内容
├── 根据学习风格推荐练习形式
├── 动态调整难度曲线（太简单→提速，太难→降级）
├── 学习计划自动生成 + 进度追踪
└── 薄弱知识点专项突破路径
```

**自进化点**：
- 追踪推荐内容的实际学习效果
- 积累"某类学生的最优学习路径"
- 路径A vs 路径B 效果对比，自动择优

## 四、学生档案系统（深度设计）

### 档案结构

```
学生档案
├── 基础信息
│   ├── 年级/学校/联系方式
│   ├── 入学时间/在读课程
│   └── 家长联系方式
│
├── 学业画像（自动采集，持续更新）
│   ├── 各知识点掌握度（0-100 热力图）
│   ├── 历次成绩趋势曲线
│   ├── 错题本（自动归类 + 错因分析）
│   ├── 薄弱知识点 + 易错题型
│   ├── 学习速度 / 进步曲线
│   └── 知识点间依赖关系（哪些前置知识缺失）
│
├── 行为画像（Agent 自动打标）
│   ├── 学习习惯标签
│   │   ├── 拖延型 — 总是卡 deadline
│   │   ├── 粗心型 — 会做但常犯低级错误
│   │   ├── 畏难型 — 遇到难题就放弃
│   │   ├── 冲刺型 — 考前突击效果好
│   │   ├── 稳定型 — 持续稳定输出
│   │   └── 自定义标签（教师手动添加）
│   ├── 学习时段偏好（上午/下午/晚上）
│   ├── 答疑互动模式（主动提问/被动回答/沉默型）
│   └── 情绪状态追踪（基于对话文本分析）
│
├── 个性化策略（AI 推荐 + 教师确认）
│   ├── 最优教学方法（基于历史效果数据）
│   ├── 推荐练习难度曲线
│   ├── 答疑引导风格
│   ├── 鼓励/督促话术模板
│   └── 学习路径规划
│
├── 沟通记录
│   ├── 与家长的沟通摘要（支持语音转文字录入）
│   ├── 教师备注
│   └── 关键事件标记（如"期中考试前焦虑"）
│
└── 长期追踪
    ├── 阶段性对比（入学 vs 3个月 vs 6个月）
    ├── 成长报告（自动生成，可发家长）
    ├── 跨课程/跨学期连续性
    └── 效果归因（哪些教学行为带来了进步）
```

### 档案的"活"机制

| 触发 | 更新内容 |
|------|---------|
| 学生做题 | 正答率、知识点掌握度、错题本自动更新 |
| AI 对话答疑 | 行为标签刷新、答疑效果评估 |
| 教师课后 30 秒 | 确认/修正 AI 建议、添加备注 |
| 每周 | 行为标签重新评估、学情趋势更新 |
| 每月 | 自动生成成长报告、策略推荐刷新 |
| 每学期 | 长期对比分析、学习路径复盘 |

### 三层进化

#### 第一层：单教师进化（最直接）

```
场景：张老师教初中数学
├── 第 1 周：AI 生成教案，张老师大幅修改（60% 改动）
├── 第 4 周：AI 学会了张老师的风格，修改量降到 30%
├── 第 12 周：AI 生成的教案基本不需要改，偶尔微调
└── 第 24 周：AI 能根据班级学情自动调整，张老师只需确认

数据积累：
├── 张老师的教案偏好（结构/难度/互动方式）
├── 张老师班级学生的知识掌握分布
├── 哪些教学方法对这个班级最有效
└── 张老师与家长沟通的风格和话术
```

**价值**：教师投入时间教会 AI → 切换成本极高 → 留存率高

#### 第二层：跨教师进化（规模效应）

```
场景：同一个机构的 5 位数学老师
├── 王老师班上"二次函数"用了类比法，效果很好
├── 系统提取这个模式，推荐给李老师
├── 李老师试用后发现也有效，标记确认
├── 模式升级为机构级教学策略
└── 新入职的赵老师直接获得这些积累

数据积累：
├── 机构级教学策略库（经多位教师验证）
├── 各知识点的最优教学方法排名
├── 不同学生类型的最佳匹配策略
└── 教师间的经验传递（无形化为有形）
```

**价值**：机构知识不随教师离职流失 → 机构依赖这个系统

#### 第三层：跨机构进化（行业壁垒）

```
场景：100 个机构在使用
├── 全国 5000 名学生的"一元二次方程"学习数据
├── 发现：A 教法比 B 教法正答率高 23%
├── 发现：先讲图像再讲公式比反过来效果好 15%
├── 这些洞察反馈给所有教师
└── 行业级教学知识图谱逐渐成型

数据积累：
├── 全国级知识点难度模型
├── 教学方法效果排名
├── 学生学习路径最优解
└── 跨学科教学模式迁移
```

**价值**：数据网络效应 → 后来者无法复制 → 行业标准

### 进化安全边界

```
可以进化：
├── 教案生成风格 → 根据教师修改记录优化
├── 练习难度预测 → 根据正答率校准
├── 答疑话术 → 根据学生反馈调整
└── 推荐策略 → 根据效果数据排序

不能进化（红线）：
├── 不改变教学大纲和课程标准
├── 不绕过教师审核自动发布内容
├── 不用学生数据训练外部模型
├── 不做未经验证的激进策略变更
└── 教师随时可以回退到 AI 默认行为
```

## 六、护城河分析

### 五层护城河

```
第 1 层：数据飞轮
  更多使用 → 更多数据 → 更好的 AI → 更多使用
  （最核心，竞争对手拿不走你的数据）

第 2 层：教师迁移成本
  教了 AI 3 个月的风格偏好 → 换平台要重新教
  （类比：你不会轻易换用了 3 年的编辑器）

第 3 层：机构知识沉淀
  机构的教学策略库、学生档案、历史数据都在系统里
  （机构换系统 = 知识资产归零）

第 4 层：网络效应
  机构内教师互相共享策略 → 机构间优秀实践传播
  （先发优势：用户越多，价值越大）

第 5 层：行业知识图谱
  跨机构积累的教学效果数据 → 行业级洞察
  （后来者即使有技术，也没有这些数据）
```

## 八、路线图

### Phase 1（第 1-3 周）：核心链路 + 档案

```
目标：教师能生成教案/练习，学生能做题，基础档案建立
├── 教师 Web 端：教案生成 + 练习生成
├── 学生 H5：做题 + 查看解析
├── 学生档案 V1：基础信息 + 成绩趋势 + 错题本
├── 学情统计：正确率趋势 + 知识点掌握度
└── 教师行为日志采集（为自进化储备数据）
```

### Phase 2（第 4-6 周）：智能答疑 + 档案深化

```
目标：学生微信端答疑，档案开始行为画像
```

## 十、风险与对策

| 风险 | 对策 |
|------|------|
| AI 生成内容质量不稳 | 教师审核不可跳过 + 质量评分淘汰 |
| 教师不愿教 AI（冷启动） | 提供开箱即用的模板，降低初始门槛 |
| 学生过度依赖 AI | 引导式设计 + 每日答疑次数限制 |
| 数据安全 | 加密存储 + 定期审计 + 合规先行 |
| API 成本上涨 | 预留模型切换 + 数据积累后考虑微调 |
| 大厂入场 | 靠数据飞轮和教师迁移成本建立壁垒 |
| 进化质量失控 | 效果验证机制 + 教师可回退 + 人工抽检 |
