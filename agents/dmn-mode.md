# DMN 空闲模式 — Default Mode Network

> 神经科学概念：大脑在"什么都不做"的时候，其实在做记忆整合和自我反思
> 对应到系统：空闲时自动进入觉知循环

## 触发条件

| 条件 | 判断方式 |
|------|---------|
| 距上次对话 >30分钟 | session 最后消息时间 |
| 无活跃 cron 任务 | cron list 检查 |
| 无进行中的子会话 | sessions_list 检查 |
| 工作时间外（23:00-08:00） | 时间判断 |

## DMN 行为

当系统进入空闲状态时，自动执行：

### 1. 快速觉知扫描（轻量级）
- 检查 memory-state.json 时间戳
- 检查 .skill-quality.json 异常
- 检查今日 daily 日志是否有未处理信号

### 2. 微整合（如果有材料）
- 识别今日日志中的重复模式
- 检查是否有待确认的 topic 条目
- 更新质量计数器

### 3. 状态报告（仅异常时通知）
- 如果发现异常 → 通知用户
- 如果一切正常 → 静默

## 与 Cron 的关系

| 机制 | 触发方式 | 频率 | 重量级 |
|------|---------|------|--------|
| Cron（02:00） | 定时 | 每日1次 | 重（完整整合） |
| Cron（23:30） | 定时 | 每日1次 | 重（完整反思） |
| DMN | 空闲 | 每次空闲 | 轻（快速扫描） |

DMN 不替代 Cron，它是**补充**。Cron 做完整处理，DMN 做快速扫描。

## 实现方式

在 HEARTBEAT.md 中加入 DMN 检查逻辑：
- 每次心跳检查是否空闲
- 空闲时执行轻量觉知扫描
- 有发现才触发完整流程
