# 架构进化 — Architecture Evolution

> 不只是进化 Skill 和记忆，循环结构本身也能进化
> 这是最高层的进化

## 进化维度

| 维度 | 当前 | 可进化方向 |
|------|------|-----------|
| 循环数量 | 3 个（觉知/执行-验证/记忆） | 可增可减 |
| 层间通讯 | 5 条固定链路 | 可动态调整 |
| 验证策略 | 每层固定验证方式 | 可自适应 |
| 频率 | Cron 固定时间 | 可根据负载调整 |

## 触发条件

| 条件 | 操作 |
|------|------|
| 某层连续空闲 >7天 | 考虑合并到其他层 |
| 某层任务积压 | 考虑拆分为子层 |
| 层间通讯瓶颈 | 考虑增加通讯链路 |
| 新功能无法归入现有层 | 考虑新增层 |

## 安全边界

架构进化是**最危险的进化**，因为它改变系统的基本结构。

| 限制 | 原因 |
|------|------|
| 不允许删除觉知循环 | 观察者是系统安全的最后防线 |
| 不允许修改 SOUL.md 核心规则 | 这是身份定义，不是可进化组件 |
| 每次架构进化必须用户确认 | 防止失控 |
| 架构进化后必须运行完整测试 | 确保系统完整性 |

## 进化日志

架构进化记录到 `.snapshots/changelog.json`，标记为 `type: architecture`。
