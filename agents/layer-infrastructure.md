# 3 层基础设施配置

> 每层有自己的记忆格式、验证方式、运行频率
> 不是所有层共用同一套东西

---

## 👁️ 觉知循环 — 基础设施

| 维度 | 配置 |
|------|------|
| **记忆格式** | 反思报告（结构化模板） |
| **存储位置** | memory/daily/YYYY-MM-DD.md 末尾 |
| **验证方式** | 模式识别（信号计数 + 趋势对比） |
| **运行频率** | 每日 1 次（23:30）+ 按需触发 |
| **数据源** | daily 日志 + .skill-quality.json + memory-state.json |
| **输出** | 觉知报告（信号/教训/模式/指导） |
| **决策权限** | 可触发 FIX/DERIVED/CAPTURED，不可直接执行 |

### 反思报告模板

```markdown
## 👁️ 觉知 HH:MM
- 执行质量: [评估]
- 记忆健康: [评估]
- 进化状态: [评估]
- 模式识别: [发现的模式]
- 指导信号: [对其他循环的指导]
```

## ⚡ 执行-验证循环 — 基础设施

| 维度 | 配置 |
|------|------|
| **记忆格式** | 执行日志（任务/工具/结果/验证） |
| **存储位置** | memory/daily/YYYY-MM-DD.md（执行段落） |
| **验证方式** | 文件验证 + 命令验证 + 状态验证 |
| **运行频率** | 实时（用户指令触发） |
| **数据源** | 用户指令 + 觉知循环指派 + Cron 任务 |
| **输出** | 执行报告（任务/执行/验证/状态） |
| **决策权限** | 可执行工具，破坏性操作需用户确认 |

### 任务分级

| 级别 | 特征 | 验证策略 |
|------|------|---------|
| S | 简单查询/读取 | 无验证 |
| A | 文件操作/搜索 | 快速验证（读取确认） |
| B | 外部调用/复杂任务 | 完整验证（结果对比） |
| C | 破坏性/不可逆 | 验证 + 用户确认 |

## 🧠 记忆整合循环 — 基础设施

| 维度 | 配置 |
|------|------|
| **记忆格式** | 4 层架构（L1索引/L2主题/L3日志/L4会话） |
| **存储位置** | MEMORY.md + topics/ + daily/ + sessions/ |
| **验证方式** | 健康指标（行数/条目数/过期率） |
| **运行频率** | 每日 3 次（02:00整合/02:15健康/09:00巡检） |
| **数据源** | daily 日志 + topic 文件 + memory-state.json |
| **输出** | 整合报告 + 健康报告 + 巡检报告 |
| **决策权限** | 可修改记忆文件，可触发进化 |

### 健康指标

| 指标 | 阈值 | 操作 |
|------|------|------|
| MEMORY.md 行数 | >150 | 预警，建议精简 |
| Topic 条目数 | >50 | 触发拆分建议 |
| 待确认标记 | >10 | 触发清理 |
| 7天 daily 数量 | 0 | 可能写入遗漏 |
| 文件更新时间 | >7天 | 标记⚠️过期 |

## 层间通讯

```
觉知 ──指导信号──→ 执行-验证
觉知 ──进化指令──→ 记忆整合
执行-验证 ──执行结果──→ 记忆整合（写入 daily）
记忆整合 ──健康报告──→ 觉知（作为观察输入）
执行-验证 ──失败上报──→ 觉知（请求指导）
```

**关键原则**：
- 每层只关心自己的数据格式
- 层间通过标准化报告通讯
- 任何层都可以独立运行
- 嵌套：觉知可以包住执行-验证循环观察
