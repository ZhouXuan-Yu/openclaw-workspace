## 11:27 — 行动 | GGOB 记忆架构 v2 升级

基于 Claude Code /dream、Hermes 双文件、agentmemory 4层管道、OpenClaw session-logs 的综合分析。

### 架构升级

| 层 | 旧 | 新 |
|----|----|----|
| L1 | MEMORY.md 内容转储（3.2KB） | MEMORY.md 纯索引（1.2KB，<200行） |
| L2 | 无结构 | `memory/topics/` 按主题拆分（preferences/projects/learnings） |
| L3 | memory/ 根目录 | `memory/daily/` 每日日志 |
| L4 | 无影响 | sessions JSONL 最后手段 |

### 具体改动（v2 → v3 升级）
1. 设计文档 `GGOB_记忆架构_v3_机制设计.md` — 核心论点：别纠结几层，打磨三条机制
2. **写入判断树** → AGENTS.md 写策略从表格升级为分类决策树
   - 明确区分：指令/纠正/决策/偏好/日常/废话，各有不同处理路径
   - 核心规则：单次不提权，纠正权重大于陈述
3. **检索精度升级** → L2 搜索加入关键词扩展（同义表达）
4. **老化淘汰表** → AGENTS.md 新增老化规则章节
   - 7天无强化→待确认→降级；30天删除；矛盾以用户最新为准
5. cron 任务重写 → 加入老化处理和矛盾检测逻辑

### 设计考量
- **核心论点**：几层不是关键，三条机制才是（写入时机/检索精度/老化淘汰）
- **网上的4层分类法**是认知科学CoALA框架，描述的是「记忆长什么样」，不解决「怎么用」的问题
- **真正让 agent 越用越智能的**是写入判断树 + 搜索扩展 + 老化淘汰这三条可执行的规则
- **agentmemory**：Windows 原生支持不完整，未来等稳定后替换 Layer 2 的 grep 搜索
