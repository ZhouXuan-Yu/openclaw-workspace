**借鉴鲁班的三个改进**：
1. **CAPTURED 增加验料**：创建新 Skill 前先过四问（真实问题/独特角度/安装理由/可验证性）

- 健康度评级：A(4分)/B(3)/C(1)/D(0)/F(-1)
- 三种触发：FIX(失败≥2)/DERIVED(纠正≥2)/CAPTURED(成功无匹配)
- 改进必须过验证门才写入 SKILL.md

## 🔧 视频号上传修复

**时间**: 01:36
**问题**: `sau tencent upload-video` 持续超时：`Locator.set_input_files: Timeout 30000ms exceeded. waiting for locator("input[type=file]")`

**排查过程**：
1. 初次以为是 hidden input 问题 → 用 JS 让 input 可见 → 无效
2. headless 模式下访问 `/platform/post/create` → 被重定向到 `/platform` (dashboard)
3. dashboard 上没有 `input[type="file"]` → 所以 locator 永远找不到

**根因**：视频号助手检测到自动化浏览器后，访问 `/platform/post/create` 会被重定向到 dashboard，而不是加载上传表单。

**修复**：`open_upload_page` 增加重定向检测——如果被重定向到 dashboard，自动点击「发表视频」按钮进入上传表单。

**教训**：
- Playwright locator 超时 ≠ 元素不可见，可能是元素根本不存在（页面被重定向）
- 排查时应该先检查 page.url 是否是预期地址，而不是假设页面加载成功
- SPA 页面的 URL 重定向是常见的反自动化手段，遇到 locator 找不到时优先检查当前 URL
