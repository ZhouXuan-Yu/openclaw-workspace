数据流 — 写入 & 检索 & 整合

📥 写入流（实时）

  • ① 用户说了什么
  • ② 判断类别：指令 / 纠正 / 决策 / 偏好 / 日常 / 废话
  • ③ 明确指令 → daily + MEMORY.md星标 + topic
  • ④ 用户纠正 → 首次 daily，第2次 → topic
  • ⑤ 决策结论 → decisions.md + projects.md
  • ⑥ 偏好信号 → preferences.md
  • ⑦ 日常讨论 → daily/今日.md
  • ⑧ 废话 → 不写

🔍 检索流（按需）

  • L1 MEMORY.md 索引已加载 → 命中topic引用？读文件
  • L2 search-memory.ps1 精确grep → 搜topics/
  • L2+ 关键词扩展（同义词） → 搜daily/
  • L3 session-logs rg+jq → 搜JSONL
  • ∅ "我查了记忆，没找到相关内容"

🔄 整合流（cron 02:00）

  • ① 读 daily/ 最近7天日志
  • ② 识别信号：纠正 / 强化 / 失败 / 肯定
  • ③ 单次不提权，≥2次 → 写入topic
  • ④ 老化：7天无引用 → 标记待确认
  • ⑤ 矛盾检测：以用户最新陈述为准
  • ⑥ 更新 MEMORY.md 索引 → 保持<200行

🔮 反射流（对话结束时）

  • ① 这次哪里做错了？ → 检测纠正信号
  • ② 提取教训 → learnings.md
  • ③ 同一教训第2次 → 写入AGENTS.md
  • ④ 什么模式在重复？ → 连续3天同类问题
  • ⑤ 提炼规则 → preferences.md
  • ⑥ 学到新知识？ → 对应topic文件

Cron 调度 — 自动化管道

memory-consolidation

每天 02:00 CST · isolated session
  1. 读 MEMORY.md + topics/*.md + daily/ 最近7天
  2. 识别信号：纠正/强化/失败/肯定/记住/决策
  3. 老化处理：7天无强化→待确认，30天→删除
  4. 矛盾检测：以最近用户陈述为准
  5. 整合写入 topic 文件
  6. 维护 MEMORY.md 索引（<200行）
  7. 输出整合摘要
  8. 记录 lastConsolidation 时间戳

memory-health-sync

每天 02:15 CST · isolated session
  1. MEMORY.md 行数检查（阈值 150）
  2. Topic 文件条目数检查（阈值 50）
  3. 待确认标记数检查（阈值 10）
  4. 最近7天 daily 日志数量
  5. 过期文件检测（>7天未更新）
  6. 输出健康度报告
  7. git add + commit + push
  8. 记录 lastHealthCheck 时间戳

memory-patrol 巡检

每天 09:00 CST · isolated session
  1. 读取 memory-state.json（凌晨任务写入的状态）
  2. lastConsolidation ≠ 今天 → 补跑整合
  3. lastHealthCheck ≠ 今天 → 补跑健康检查
  4. 补跑 = 完整执行对应任务的所有步骤
  5. Git 同步（如有变更）
  6. 更新 lastPatrol 时间戳
  7. 输出巡检报告（✅已执行 / ⚠️已补跑）
